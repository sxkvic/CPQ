# CPQ 前端页面设计

## 1. 菜单结构

建议主菜单：

```text
工作台
客户管理
商机管理
产品管理
配置规则
价格管理
报价管理
审批中心
订单管理
报表看板
系统设置
```

第一版可以把报表看板做得简单一些，重点先保证报价流程顺畅。

## 2. 页面布局

建议采用企业后台常见布局：

- 左侧菜单。
- 顶部显示搜索、通知、当前用户。
- 中间为主内容区。
- 新建和编辑可使用抽屉或弹窗。
- 详情页使用标签页承载不同信息。

报价编辑页建议布局：

```text
报价头部
  客户、联系人、商机、有效期、状态

报价工作区
  左侧：产品搜索和分类筛选
  中间：报价明细表
  右侧：金额汇总和审批提示

底部操作
  保存草稿、重新计算、提交审批、导出、标记已发送
```

## 4. 前端状态设计

服务端状态：

- 客户
- 产品
- 价格本
- 报价
- 审批

建议使用 TanStack Query 管理接口加载、缓存和刷新。

本地状态：

- 当前报价编辑草稿
- 产品配置抽屉状态
- 未保存的明细行修改
- 表格筛选条件

复杂报价编辑状态可使用 Zustand。

## 5. 报价编辑交互

### 5.1 新建报价

1. 用户选择客户。
2. 系统加载联系人和商机。
3. 用户选择价格本。
4. 系统创建报价草稿。
5. 用户进入报价编辑页。

### 5.2 添加产品

1. 用户搜索产品。
2. 用户选择产品。
3. 如果产品存在选项，则打开配置抽屉。
4. 用户选择选项。
5. 前端调用配置校验接口。
6. 校验通过后添加报价明细。
7. 系统重新计算报价。

### 5.3 提交审批

1. 用户点击提交审批。
2. 前端先调用重新计算接口。
3. 后端返回是否需要审批和触发原因。
4. 用户确认提交。
5. 后端创建审批任务。
6. 报价状态变为待审批。

### 5.4 修改已批准报价

已批准和已发送报价不允许直接修改。

如果需要修改：

1. 点击创建新版本。
2. 系统复制当前报价快照。
3. 新版本状态为草稿。

## 6. 公共组件建议

- `MoneyText`：金额显示
- `StatusTag`：通用状态标签
- `UserSelect`：用户选择
- `CustomerSelect`：客户选择
- `ProductSelect`：产品选择
- `PriceBookSelect`：价格本选择
- `QuoteStatusTag`：报价状态标签
- `ApprovalTimeline`：审批时间线
- `ProductConfigDrawer`：产品配置抽屉
- `QuoteSummaryPanel`：报价汇总面板
- `EditableQuoteItemTable`：可编辑报价明细表
- `AuditLogTable`：操作日志表

## 7. 体验原则

- 明细变化后尽快展示计算结果。
- 始终显示当前报价状态。
- 不可执行的状态动作要禁用。
- 审批原因必须用业务人员看得懂的话说明。
- 离开报价编辑页前，提醒未保存内容。
- 金额字段要对齐并统一格式。
- 如果时间允许，导出前提供报价单预览。
