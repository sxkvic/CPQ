# CPQ 交付路线图

## 1. 交付策略

建议分三阶段建设：

```text
第一阶段：能创建、计算、审批、导出的 MVP
第二阶段：增强产品配置、价格策略和文档能力
第三阶段：系统集成、报表分析和高级自动化
```

第一阶段最重要的是报价闭环和价格准确性。复杂规则引擎可以后置。

## 2. 第一阶段 MVP

一个人前后端开发，预计周期 6 到 8 周。

### 范围

- 项目脚手架
- 登录和权限
- 客户管理
- 联系人管理
- 产品分类和产品管理
- 基础产品选项
- 价格本和价格明细
- 新建报价
- 报价明细管理
- 后端报价计算
- 手工折扣
- 审批触发判断
- 一到两级审批流程
- 报价 PDF 导出
- 报价版本快照
- 基础操作日志

### 验收标准

- 销售可以创建客户和联系人。
- 产品经理可以维护产品和选项。
- 价格经理可以维护价格本。
- 销售可以新建报价并添加产品。
- 系统可以计算小计、折扣、税额、总价、成本和毛利。
- 系统可以阻止错误的产品配置。
- 折扣或毛利触发规则时，系统要求审批。
- 审批人可以通过或驳回报价。
- 已批准报价可以导出 PDF。
- 已批准报价不能直接编辑。
- 用户可以基于已批准报价创建新版本。

## 3. 第二阶段增强

预计周期 4 到 6 周。

### 范围

- 套餐产品支持
- 数量阶梯价格
- 更灵活的配置规则
- 报价 Excel 导出
- 报价模板管理
- 审批时间线和评论
- 报价转订单
- 报价即将过期提醒
- Excel 导入产品和价格
- 更完整的工作台看板

### 验收标准

- 套餐产品可以包含必选和可选子产品。
- 数量变化可以匹配阶梯价。
- 配置规则可以表达依赖和互斥。
- 报价可以导出 Excel。
- 客户接受的报价可以转为订单。
- 用户可以通过 Excel 导入产品和价格。

## 4. 第三阶段高级能力

预计周期 6 到 10 周。

### 范围

- CRM 集成
- ERP 集成
- 多币种和汇率
- 渠道伙伴报价
- 高级毛利分析
- 合同生成
- 电子签章集成
- API Webhook
- 可配置审批流程设计器
- 规则引擎升级

## 5. 开发里程碑

### 里程碑 1：基础框架

交付内容：

- 前后端项目脚手架
- 数据库模型和迁移
- 登录

- 产品选项

### 里程碑 3：报价核心

交付内容：

- 报价明细表

### 里程碑 4：审批和导出

交付内容：

交付内容：

## 6. 风险清单

| 风险 | 影响 | 处理方式 |
| --- | --- | --- |
| 产品规则一开始就过于复杂 | 拖慢 MVP | 第一版只做固定类型规则 |
| 价格计算散落在多个地方 | 结果不一致 | 后端计算服务作为唯一依据 |
| 已审批报价还能被编辑 | 商务和法务风险 | 使用状态锁定和报价快照 |
| 审批规则不明确 | 返工 | 先确定折扣、毛利、金额阈值 |
| PDF 模板频繁变化 | 返工 | 模板单独维护，便于替换 |
| 产品数据质量差 | 报价错误 | 增加必填校验和导入校验 |

## 7. 演示数据建议

客户：

- ABC制造有限公司
- 星河能源有限公司
- 都市自动化有限公司

产品：

- 标准设备1000
- 高级设备2000
- 安装实施服务
- 年度维保服务
- 远程监控模块

选项：

- 功率等级：标准、高功率
- 散热方案：标准散热、增强散热
- 质保年限：1年、2年、3年
- 监控方案：无、基础版、高级版

规则：

- 高功率必须选择增强散热。
- 高级监控必须购买年度维保服务。
- 折扣超过 20% 需要销售经理审批。
- 毛利率低于 15% 需要财务审批。

## 8. 第一轮开发任务建议

1. 创建项目结构。
2. 初始化后端项目。
3. 初始化前端项目。
4. 创建数据库模型。
5. 实现登录。
6. 实现后台基础布局。
7. 实现客户列表和表单。
8. 实现产品列表和表单。
9. 实现价格本列表和表单。
10. 准备演示用户、客户、产品和价格数据。

## 9. 完成定义

一个功能完成时，应满足：

- API 有参数校验和权限校验。
- 前端处理加载、空数据、错误和成功状态。
- 数据能正确保存。
- 敏感操作有操作日志。
- 主要流程经过人工测试。
- 有相关演示数据或测试数据。
- 页面文案和状态名称业务人员能看懂。

## 10. 一个人开发的建议

- MVP 流程要收窄，别一开始铺太大。
- 第一版不要做完全通用的规则引擎。
- 报价计算服务要写清楚，并重点测试。
- 多使用 Ant Design 的表格、表单、抽屉、弹窗和步骤条。
- 配置规则和报价快照可以合理使用 PostgreSQL JSON 字段。
- 报价编号和版本号必须后端生成。
- PDF 导出按模板生成，不依赖浏览器打印。
