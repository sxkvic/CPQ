# CPQ 系统总览

## 1. 系统定位

CPQ 系统是面向销售和销售管理团队的内部报价平台。系统用于规范产品配置、价格计算、折扣控制、报价审批、报价单导出和报价版本管理。

第一版建议按制造业、设备销售或解决方案销售场景设计。该模型比较完整，后续也容易改造成软件授权、服务报价、项目报价或渠道报价。

## 2. 建设目标

- 降低人工报价工作量。
- 减少产品配置错误。
- 统一价格计算口径。
- 控制折扣和低毛利风险。
- 缩短报价审批周期。
- 保留完整报价版本和审批记录。
- 支撑 CRM、ERP、合同、订阅、文档和订单系统集成。

## 3. 使用角色

| 角色 | 主要职责 |
| --- | --- |
| 销售 | 新建报价、选择客户、配置产品、申请折扣、导出报价 |
| 销售经理 | 审核团队报价和折扣申请 |
| 产品经理 | 维护产品、选项、套餐、配置规则 |
| 价格经理 | 维护价格本、折扣规则、税率规则 |
| 财务 | 审核毛利、税费、特殊价格 |
| 管理员 | 管理用户、角色、权限、系统参数 |
| 高层审批人 | 审批大金额或特殊风险报价 |

## 4. 核心流程

```text
客户 / 商机
  -> 新建报价
  -> 选择产品
  -> 配置选项
  -> 校验配置规则
  -> 计算价格
  -> 设置折扣 / 税费 / 其他费用
  -> 提交审批
  -> 生成报价单
  -> 发送客户
  -> 客户接受 / 拒绝 / 报价过期
  -> 转订单
  -> 生成合同
  -> 订阅生效 / 续费跟进
  -> 同步 ERP / CRM / 财务系统
```

## 5. 系统范围

### 第一版范围

- 用户登录和权限
- 客户、联系人、商机管理
- 产品目录
- 产品选项和套餐
- 基础配置规则校验
- 价格本
- 报价计算
- 折扣申请
- 审批流程
- 报价单导出和文档模板
- 报价版本记录
- 操作日志
- 合同管理
- 订阅与续费管理
- 出站集成事件队列

### 后续范围

- 可视化产品配置器
- 复杂规则引擎
- 多币种与汇率
- 渠道伙伴报价
- 电子签章
- 高级利润分析
- AI 辅助报价推荐

## 6. 核心业务对象

| 对象 | 说明 |
| --- | --- |
| 客户 | 接收报价的公司或组织 |
| 联系人 | 客户下的具体联系人 |
| 商机 | 销售机会，可关联多个报价版本 |
| 产品 | 可销售的设备、软件、服务或方案 |
| 选项 | 产品可配置的参数或配件 |
| 套餐 | 由多个产品组成的打包方案 |
| 价格本 | 按区域、客户等级、渠道或时间定义的价格策略 |
| 报价 | 给客户的商业报价 |
| 报价明细 | 报价中的单个产品行 |
| 审批 | 折扣、毛利、金额等审批记录 |
| 订单 | 客户接受报价后形成的执行记录 |
| 文档模板 | 报价单或合同输出所使用的 HTML 模板 |
| 报价文档 | 已生成的报价文档快照 |
| 合同 | 由报价或订单形成的商业协议 |
| 订阅 | 软件或服务的周期性权益、账期和续费记录 |
| 集成端点 | 外部 ERP、CRM、财务系统的连接定义 |
| 集成事件 | 等待发送、已发送或失败重试的出站事件 |

## 7. 设计原则

- 报价页面是系统核心。
- 产品和价格维护要让业务人员能理解。
- 每一个价格结果都应能解释。
- 每一个审批都要显示触发原因。
- 已审批或已发送报价必须锁定。
- 修改已确认报价时，应生成新版本。
- 第一版优先做清晰流程，不急于做完全通用的规则引擎。
