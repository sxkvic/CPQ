# CPQ 需求规格说明

## 1. 功能模块

### 1.1 登录与权限

- 用户登录和退出。
- 支持管理员初始化或重置密码。
- 基于角色控制菜单和按钮权限。
- 可选支持部门或团队数据范围。
- 关键操作记录审计日志。

建议角色：

| 角色 | 权限概述 |
| --- | --- |
| 管理员 | 管理所有系统配置 |
| 销售 | 管理自己的客户、商机和报价 |
| 销售经理 | 管理团队报价和审批任务 |
| 产品经理 | 管理产品、选项、套餐、配置规则 |
| 价格经理 | 管理价格本、折扣、税率 |
| 财务 | 查看财务字段并审批毛利相关事项 |
| 高层审批人 | 审批大金额或特殊报价 |

### 1.2 客户管理

- 新建、编辑、禁用、查询客户。
- 管理客户联系人。
- 指定客户负责人。
- 维护客户等级、行业、区域、税号、开票地址。
- 查看客户历史报价。

### 1.3 商机管理

- 在客户下新建商机。
- 维护销售阶段、预计成交日期、预计金额、负责人。
- 一个商机可以关联多份报价或多个报价版本。
- 成交报价可转为订单。

### 1.4 产品管理

- 管理产品分类。
- 管理产品和 SKU。
- 支持设备、软件、服务、实施、套餐等类型。
- 产品字段包括：
  - 产品编码
  - 产品名称
  - 产品分类
  - 产品描述
  - 单位
  - 标准成本
  - 标准售价
  - 状态
  - 生效日期
  - 失效日期

### 1.5 产品配置

- 管理产品选项。
- 支持必选项和可选配件。
- 支持选项值加价或减价。
- 支持套餐产品。
- 支持基础校验：
  - 必选项未选择
  - 选项互斥
  - 选项依赖
  - 最小或最大数量
  - 区域或客户限制

### 1.6 价格管理

- 按区域、客户类型、渠道或时间维护价格本。
- 维护产品价格和选项价格。
- 支持：
  - 标准价
  - 客户等级价
  - 阶梯价
  - 选配加价
  - 手工折扣
  - 自动折扣
  - 税率
  - 服务费
  - 运费

### 1.7 报价管理

- 从客户或商机新建报价。
- 添加产品和套餐。
- 配置产品选项。
- 计算行金额和总金额。
- 支持明细行折扣和整单折扣。
- 保存草稿。
- 提交审批。
- 生成客户报价单。
- 复制报价。
- 创建新版本。
- 标记为已接受、已拒绝、已取消、已过期。

报价状态建议：

```text
草稿 -> 待审批 -> 已批准 -> 已发送 -> 已接受 -> 已转订单
草稿 -> 已取消
待审批 -> 已驳回 -> 草稿
已批准 -> 已过期
已发送 -> 已拒绝
```

### 1.8 报价计算

明细行计算：

```text
基础金额 = 单价 * 数量
选项金额 = 所有选项价格调整之和
行标价金额 = 基础金额 + 选项金额
行折扣金额 = 行标价金额 * 行折扣率
行净额 = 行标价金额 - 行折扣金额
```

整单计算：

```text
小计 = 所有行净额之和
整单折扣金额 = 小计 * 整单折扣率
应税金额 = 小计 - 整单折扣金额 + 服务费 + 运费
税额 = 应税金额 * 税率
总金额 = 应税金额 + 税额
毛利 = 总金额 - 总成本
毛利率 = 毛利 / 总金额
```

### 1.9 审批流程

第一版审批规则建议简单可配置。

示例：

| 触发条件 | 审批人 |
| --- | --- |
| 任一明细行折扣超过 20% | 销售经理 |
| 整单折扣超过 15% | 销售经理 |
| 整单折扣超过 30% | 高层审批人 |
| 毛利率低于 15% | 财务 |
| 报价总金额超过 100 万 | 高层审批人 |
| 手工改价 | 价格经理 |

审批动作：

- 提交
- 通过
- 驳回
- 撤回
- 重新提交

审批记录应包含：

- 触发原因
- 当前审批人
- 审批层级
- 审批意见
- 操作时间
- 操作前后状态

### 1.10 文档导出

第一版支持：

- PDF 报价单
- Excel 报价明细

报价单内容：

- 公司 Logo 和销售方信息
- 客户名称和联系人
- 报价编号和版本号
- 报价有效期
- 产品明细
- 数量、单价、折扣、税额、总价
- 付款条款
- 交付条款
- 备注
- 签字盖章区域

### 1.11 转订单

- 客户接受的报价可以转为订单。
- 转订单后的报价应锁定。
- 订单继承客户、联系人、报价明细、价格和条款。
- 后续可对接 ERP。

### 1.12 操作日志

建议记录以下操作：

- 登录失败
- 修改价格
- 修改产品规则
- 提交报价
- 审批报价
- 手工改价
- 导出报价
- 修改报价状态

## 2. 非功能需求

### 2.1 性能

- 常用筛选下，报价列表 2 秒内加载完成。
- 普通报价计算 1 秒内完成。
- 产品查询支持关键字和分类筛选。

### 2.2 安全

- 密码必须加密存储。
- Token 需要过期机制。
- 敏感接口必须校验角色和数据权限。
- 没有权限的用户不显示成本和毛利字段。

### 2.3 可维护性

- 报价计算统一放在后端服务。
- 审批触发条件应可配置。
- 产品规则尽量数据化。
- 报价模板应可替换。

### 2.4 可追溯性

- 已审批和已发送报价不能直接编辑。
- 修改报价应生成新版本。
- 报价快照和审批记录必须长期保留。
