# 开源参考与集成

本项目会优先选择许可证清晰、可商用或适合内部系统使用的开源库。对于完整业务系统类项目，只参考产品结构和功能设计，不直接复制代码。

## 已集成

| 名称 | 用途 | 许可证 | 集成方式 |
| --- | --- | --- | --- |
| json-rules-engine | 后端审批触发规则判断 | ISC | 用于折扣、毛利率、报价金额审批判断 |
| html2pdf.js | 前端报价单 PDF 下载 | MIT | 将后端生成的报价 HTML 转为 PDF |

## 参考项目

| 项目 | 可参考点 | 处理方式 |
| --- | --- | --- |
| code-boxx/quotr | 报价单 HTML/PDF/DOCX/PNG 输出思路 | 只参考报价文档能力边界 |
| Impre-visible/invoicerr | React + NestJS + Prisma 的报价/发票系统结构 | AGPL-3.0，不复制代码，只参考功能模块 |
| firxworx/react-simple-invoice | 动态明细行和实时合计交互 | MIT，可参考交互思路 |
| puckeditor/puck | 后续报价模板可视化编辑 | MIT，暂不集成，作为模板设计器候选 |

## 当前选型说明

第一阶段优先集成两个低风险能力：

- `json-rules-engine`：让审批触发条件从硬编码逐步过渡到 JSON 规则。
- `html2pdf.js`：让当前 HTML 报价单先具备 PDF 下载能力。

后续如果要做“报价模板可视化设计器”，可以评估集成 `puckeditor/puck`。如果要做更专业的服务端 PDF，可以评估 Puppeteer 或 React PDF。
