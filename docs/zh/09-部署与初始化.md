# 部署与初始化说明

## 1. 本地预览模式

当前机器如果没有 PostgreSQL 或 Docker，可以使用 Mock API 预览页面。

```powershell
powershell -ExecutionPolicy Bypass -File scripts/dev.ps1
```

访问：

```text
http://localhost:5173
```

停止：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/stop-dev.ps1
```

## 2. 真实数据库模式

需要准备 PostgreSQL，并配置根目录 `.env`：

```text
DATABASE_URL="postgresql://cpq:cpq_password@localhost:5432/cpq?schema=public"
JWT_SECRET="replace-with-a-long-random-secret"
JWT_EXPIRES_IN="1d"
API_PORT="3000"
WEB_PORT="5173"
```

初始化数据库：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/init-db.ps1
```

启动真实后端和前端：

```powershell
npm.cmd run dev -w apps/api
npm.cmd run dev -w apps/web
```

## 3. Docker Compose 部署

如果服务器安装了 Docker：

```bash
docker compose up -d --build
```

访问：

```text
http://localhost:8080
```

服务：

- `cpq-db`: PostgreSQL
- `cpq-api`: NestJS API
- `cpq-web`: Nginx + React 静态站点

## 4. 健康检查

本地预览检查：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/health.ps1
```

预期：

```text
Web 5173: True
API 3000: True
```
